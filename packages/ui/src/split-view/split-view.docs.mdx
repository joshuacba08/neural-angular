# Split View

`NSplitView` is the resizable panel layout pattern for app workspaces. It provides horizontal or vertical split panes and serves as the base for queue, preview, timeline, and inspector compositions in the design system.

## Features

### Import

```ts
import { NSplitView } from '@neuralangular/angular-ui';
```

> **v0.1.x** — Import from the root entry `@neuralangular/angular-ui`. Per-component subpath imports are planned for a future release.

### Horizontal Split

The draggable two-pane layout from the design system: video preview on the left, inspector on the right, with a 4px resize gutter.

<Canvas of={SplitViewStories.HorizontalSplit} layout="padded" sourceState="shown" />

### Three-Pane Workspace

The compact queue · preview · inspector composition uses fixed pane widths. Combine it with nested split views when you need resizable preview and inspector regions.

<Canvas of={SplitViewStories.ThreePane} layout="padded" sourceState="shown" />

### Resizable Workspace

Nested split views recreate the full Oroya-style workspace with a draggable queue rail and preview/inspector divider.

## Theming

Classes include `.n-split-view`, `.n-split-view--horizontal`, `.n-split-view--vertical`, `.n-split-view__panel`, `.n-split-view__gutter`, `.n-split-view__gutter--active`, and `.n-split-view__gutter-handle`.

The split view consumes border, transition, and surface tokens while leaving panel content styling to the composed app surface.
