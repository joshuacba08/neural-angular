## Storybook

Run the UI library documentation locally:

```bash
pnpm nx run @neuralangular/angular-ui:storybook
```

Storybook is the right place for:

- isolated component states
- design-token inspection
- visual review of variants and sizes
- custom MDX documentation

## Playground App

Run the composed Angular application locally:

```bash
pnpm nx serve playground
```

The workspace playground is the right place for:

- validating multiple components together
- checking shell and navigation ergonomics
- trying realistic pages and data density
- testing global provider behavior

## Recommended Workflow

<div className="n-guide-grid">
  <article className="n-guide-card">
    <strong>1. Start in Storybook</strong>
    <p>Dial in variants, spacing, and interaction states while the component surface is small.</p>
  </article>
  <article className="n-guide-card">
    <strong>2. Move to playground</strong>
    <p>Compose the component with cards, shell, forms, and content hierarchy that resembles a product screen.</p>
  </article>
  <article className="n-guide-card">
    <strong>3. Fold learnings back</strong>
    <p>Promote stable patterns into stories and docs so the design language keeps getting sharper.</p>
