## Overview

### What the system covers

The token layer is responsible for:

- semantic color roles
- deep surface hierarchy
- typography and mono metadata styles
- spacing and geometry
- motion timing and easing
- glow, shadow, and elevation depth

### Guidelines

<div className="n-doc-table-wrap">
  <table>
    <thead>
      <tr>
        <th>Principle</th>
        <th>Guideline</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>Use semantic roles first</td>
        <td>Prefer tokens like <code>--n-color-success</code> over hardcoded hex values inside components.</td>
      </tr>
      <tr>
        <td>Preserve the surface ladder</td>
        <td>Move between <code>canvas → base → surface 1–4</code> rather than inventing one-off panel colors.</td>
      </tr>
      <tr>
        <td>Keep glow intentional</td>
        <td>Reserve glow tokens for premium emphasis, focus states, and AI-centric accents.</td>
      </tr>
      <tr>
        <td>Match text to contrast</td>
        <td>Use <code>text-1</code> for primary information, then step down to <code>text-2/3/4</code> for hierarchy.</td>
      </tr>
      <tr>
        <td>Favor token composition</td>
        <td>Components should compose from shared tokens instead of introducing private visual constants unless truly necessary.</td>
      </tr>
    </tbody>
  </table>
