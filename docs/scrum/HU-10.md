## Historia de Usuario

Como desarrollador de Neural Angular,
quiero tener una capa de motion segura, reutilizable y compatible con SSR,
para aplicar animaciones modernas en componentes y pantallas sin usar GSAP directamente en cada componente ni romper hidratación/renderizado server-side.

---

## Objetivo principal

Crear la primera fundación de motion para `@neural/angular-ui`.

Al terminar esta HU, el paquete debe tener:

* Provider de motion.
* Servicio central de motion.
* Configuración global.
* Soporte para `prefers-reduced-motion`.
* Helpers GSAP-safe.
* Directiva simple para animaciones declarativas.
* Directiva simple para reveal on init/viewport.
* CSS utilities de motion.
* Demos en playground.
* Documentación.

# 1. Dependencia permitida

Se permite instalar GSAP.

Si `gsap` ya está instalado, reutilizarlo.

Si no está instalado, instalar:

```bash
pnpm add gsap
```

Actualizar `packages/ui/package.json`.

Estrategia recomendada para `@neural/angular-ui`:

```json
{
  "peerDependencies": {
    "@angular/core": "^22.0.0",
    "@angular/common": "^22.0.0",
    "gsap": "^3.0.0"
  }
}
```

Si el workspace necesita `gsap` como `devDependency` para compilar, agregarlo donde corresponda.

No instalar librerías extra de animation.

No instalar Framer Motion, Anime.js, Motion One ni similares.

```ts
export type NeuralMotionPreference =
```

# 6. Crear NeuralMotionService

Crear:

```txt
packages/ui/src/motion/motion.service.ts
```

Debe ser:

```ts
@Injectable({ providedIn: 'root' })
export class NeuralMotionService {
  ...
}
```

Responsabilidades:

* Leer configuración global.
* Detectar si está en browser.
* Detectar reduced motion.
* Resolver si motion está habilitado.
* Cargar GSAP de forma segura.
* Proveer métodos simples de animación.

API mínima sugerida:

```ts
isBrowser(): boolean;

prefersReducedMotion(): boolean;

isMotionEnabled(): boolean;

animate(
  element: Element | Element[],
  options?: NeuralMotionOptions
): Promise<void>;

reveal(
  element: Element | Element[],
  options?: NeuralMotionOptions
): Promise<void>;

getPresetVars(
  preset: NeuralMotionPreset
): {
  from: Record<string, unknown>;
  to: Record<string, unknown>;
};
```

Reglas:

* No usar `window` directamente sin platform guard.
* Si necesita `matchMedia`, protegerlo con browser guard.
* Si motion está disabled/reduced, no animar; aplicar estado final.
* Si GSAP no está disponible o `useGsap=false`, usar fallback CSS/clase o aplicar estado final.
* No lanzar errores si GSAP no puede cargar.
* No bloquear SSR.

# 8. Crear NMotionDirective

Crear:

```txt
packages/ui/src/motion/motion.directive.ts
```

Selector sugerido:

```html
[nMotion]
```

API:

```ts
@Input('nMotion') preset: NeuralMotionPreset | '' = 'fade-up';
@Input() nMotionTrigger: NeuralMotionTrigger = 'init';
@Input() nMotionDuration?: number;
@Input() nMotionDelay?: number;
@Input() nMotionEase?: NeuralMotionEasing;
@Input() nMotionDisabled = false;
```

Uso:

```html
<n-card
  nMotion="fade-up"
  [nMotionDelay]="100"
>
  Animated card
</n-card>
```

Reglas:

* Si trigger es `init`, animar después de render en browser.
* Si trigger es `visible`, usar IntersectionObserver solo en browser.
* Si trigger es `manual`, no animar automáticamente.
* No usar IntersectionObserver sin browser guard.
* Limpiar observer al destruir.
* No animar en SSR.
* Respetar reduced motion.
* No usar `setTimeout` salvo que sea estrictamente necesario.
* Si se usa, limpiar correctamente.


Ejemplos:

```css
.n-motion-fade-in {
  animation: n-fade-in var(--n-duration-base) var(--n-ease-standard) both;
}

.n-motion-fade-up {
  animation: n-fade-up var(--n-duration-base) var(--n-ease-decelerate) both;
}

.n-motion-scale-in {
  animation: n-scale-in var(--n-duration-base) var(--n-ease-standard) both;
}

@keyframes n-fade-in {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

@keyframes n-fade-up {
  from {
    opacity: 0;
    transform: translateY(12px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes n-scale-in {
  from {
    opacity: 0;
    transform: scale(0.96);
  }
  to {
    opacity: 1;
    transform: scale(1);
  }
}

@media (prefers-reduced-motion: reduce) {
  .n-motion-fade-in,
  .n-motion-fade-up,
  .n-motion-scale-in {
    animation: none;
  }
}
```

Agregar solo utilidades necesarias.

No crear una mini Tailwind.
