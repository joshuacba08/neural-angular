# HU-003 — Core UI primitives + Lucide integration

## Contexto

El repositorio `neural-angular` ya existe.

Ya se completaron y comitearon:

* Fase 0: análisis técnico del design system importado.

## Objetivo principal

Crear una primera capa Core UI más completa para `@neural/angular-ui`:

1. Instalar e integrar `@lucide/angular`.
2. Crear `NIcon` usando Lucide de forma Angular-native.
3. Crear `provideNeuralIcons`.
4. Crear un set inicial de iconos registrados.
5. Crear `NBadge`.
6. Crear `NChip`.
7. Actualizar playground con demos reales y composición.
8. Actualizar documentación.

## Decisión técnica

Usar:

```bash
pnpm add @lucide/angular
```

No usar:

```txt
window.lucide
lucide global script
unpkg script
manual DOM replacement
document.querySelector para iconos
```

La integración debe usar `@lucide/angular`.

---

## Reglas importantes

1. Sí se permite instalar `@lucide/angular`.
2. No usar scripts globales.
3. No usar `window.lucide`.
4. No usar `lucide.createIcons()`.
5. No usar manipulación manual de DOM.
6. No integrar GSAP todavía.
7. No crear todos los componentes del design system.
8. No crear Angular Material integration todavía.

# 4. Crear provider de iconos

Crear:

```txt
packages/ui/src/icon/icon-provider.ts
```

Debe exponer:

```ts
export function provideNeuralIcons(): Provider[];
```

Internamente debe usar `provideLucideIcons`.

Ejemplo orientativo:

```ts
import { makeEnvironmentProviders } from '@angular/core';
import { provideLucideIcons } from '@lucide/angular';
import { NEURAL_LUCIDE_ICONS } from './neural-icons';

export function provideNeuralIcons() {
  return makeEnvironmentProviders([
    provideLucideIcons(...NEURAL_LUCIDE_ICONS),
  ]);
}
```

Si `provideLucideIcons` devuelve un tipo distinto, adaptar correctamente.

Debe poder usarse así:

```ts
import { provideNeuralTheme, provideNeuralIcons } from '@neural/angular-ui';

export const appConfig = {
  providers: [
    provideNeuralTheme({ defaultTheme: 'dark' }),
    provideNeuralIcons()
  ]
};
```

# 5. Implementar NIcon con Lucide

Crear componente:

```ts
NIcon
```

Selector:

```html
<n-icon></n-icon>
```

Debe usar internamente `LucideDynamicIcon` de `@lucide/angular`.

## API mínima

Tipos:

```ts
export type NIconSize = 'xs' | 'sm' | 'md' | 'lg' | 'xl';
```

Inputs:

```ts
name: string;
size: NIconSize;
label?: string;
decorative: boolean;
strokeWidth: number;
```

Defaults:

```ts
name = '';
size = 'md';
decorative = true;
strokeWidth = 2;
```

Uso esperado:

```html
<n-icon name="sparkles" />

<n-icon name="settings" size="lg" />

<n-icon
  name="trash-2"
  size="sm"
  label="Delete"
  [decorative]="false"
/>
```

## Template esperado

Debe renderizar un SVG Lucide dinámico.

Ejemplo conceptual:

```html
<svg
  [lucideIcon]="name"
  [attr.aria-hidden]="decorative ? 'true' : null"
  [attr.role]="decorative ? null : 'img'"
  [attr.aria-label]="!decorative ? label : null"
  [attr.stroke-width]="strokeWidth"
></svg>
```

Adaptar a la API real de `@lucide/angular`.

## Styling

`NIcon` debe:

* Usar `display: inline-flex`.
* Usar `currentColor`.
* Tener tamaños por token/CSS var.
* Alinear bien con texto, botones, badges y chips.

Tamaños:

```txt
xs = 12px
sm = 16px
md = 20px
lg = 24px
xl = 32px
```

Agregar tokens si no existen:

```css
--n-icon-size-xs: 12px;
--n-icon-size-sm: 16px;
--n-icon-size-md: 20px;
--n-icon-size-lg: 24px;
--n-icon-size-xl: 32px;
```

Crear estructura:

```txt

├─ badge.component.html
├─ badge.component.scss
├─ badge.types.ts
└─ index.ts
```

Selector:

```html
<n-badge></n-badge>
```

## API

```ts
export type NBadgeVariant =
  | 'neutral'
  | 'primary'
  | 'secondary'
  | 'success'
  | 'warning'
  | 'danger'
  | 'info';

export type NBadgeSize = 'sm' | 'md';

export type NBadgeShape = 'pill' | 'square';
```

Inputs:

```ts
variant: NBadgeVariant;
size: NBadgeSize;
shape: NBadgeShape;
dot: boolean;
```

Defaults:

```ts
variant = 'neutral';
size = 'md';
shape = 'pill';
dot = false;
```

Uso:

```html
<n-badge>Default</n-badge>
<n-badge variant="success" [dot]="true">Ready</n-badge>
<n-badge variant="danger">Error</n-badge>
<n-badge variant="warning" size="sm">Pending</n-badge>
```

Debe usar tokens `--n-*`.

Si faltan tokens alpha, agregar:

```css
--n-color-primary-alpha-10
--n-color-secondary-alpha-10
--n-color-tertiary-alpha-10
--n-color-success-alpha-10
--n-color-warning-alpha-10
--n-color-danger-alpha-10
```

# 7. Implementar NChip

Crear estructura:

```txt
packages/ui/src/chip/
├─ chip.component.ts
├─ chip.component.html
├─ chip.component.scss
├─ chip.types.ts
└─ index.ts
```

Selector:

```html
<n-chip></n-chip>
```

## API

```ts
export type NChipVariant = 'default' | 'primary' | 'secondary';
export type NChipSize = 'sm' | 'md';
```

Inputs:

```ts
variant: NChipVariant;
size: NChipSize;
selected: boolean;
disabled: boolean;
removable: boolean;
```

Output:

```ts
removed: EventEmitter<void>
```

O usar output funcional si el proyecto ya está usando esa convención.

Defaults:

```ts
variant = 'default';
size = 'md';
selected = false;
disabled = false;
removable = false;
```

Uso:

```html
<n-chip>Default</n-chip>
<n-chip variant="primary" [selected]="true">Angular</n-chip>
<n-chip variant="secondary">Lucide</n-chip>
<n-chip [disabled]="true">Disabled</n-chip>
<n-chip [removable]="true" (removed)="onRemove()">Removable</n-chip>
```

## Remove action

Si `removable` es true:

* Mostrar botón interno.
* El botón debe ser `<button type="button">`.
* Debe usar `NIcon` con `name="x"` si es viable.
* Debe tener `aria-label="Remove chip"`.
* No emitir `removed` si `disabled` es true.
